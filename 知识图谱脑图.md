#### 机器学习阶段
- 机器学习阶段：感知机、逻辑回归、SVM、KNN、决策树、随机森林、GBDT、XGBoost、LightGBM的局限性

其中 感知机 是历史上神经网络的雏形，但因为它是单层结构、无法解决异或问题，现在学界普遍把它归为传统机器学习中的线性模型

特点：端到端学习深层特征、层数深、需要大量数据与算力、擅长图像/语音/文本

#### Transformer核心架构
##### 整体结构
- **编码器（Encoder）**：多层自注意力 + 前馈网络
- **解码器（Decoder）**：掩蔽自注意力 + 编码器-解码器注意力

##### 自注意力机制
- **计算步骤**：Q/K/V 矩阵生成 → 注意力分数计算 → Softmax 归一化
- **多头注意力**：并行注意力头的作用（捕捉不同特征）

##### 单词表示
- **Tokenization**：子词分割（BPE 算法）
- **位置编码**：绝对位置编码与相对位置编码的区别

##### 预测逻辑
- **自回归生成**：从左到右的 token 预测流程
- **上下文理解**：双向注意力的信息融合

#### Transformer变体

##### BERT 架构
- **预训练任务**：Masked LM（掩码语言模型）、Next Sentence Prediction
- **双向编码优势**：适合理解类任务（问答/分类）

##### GPT 架构
- **仅解码器结构**：自回归生成的效率
- **预训练任务**：因果语言模型（CLM）

##### MoE 模型（混合专家模型）
- **工作原理**：路由机制（Router）选择专家层（Expert）
- **优势**：参数量扩展但计算量可控
- **应用场景**：大参数量模型（如 GPT-4、Gemini）

## 六、大模型微调实战

### （一）微调基础

- **定义**：在预训练模型基础上，用特定数据集继续训练

#### 1. 核心概念

- **与 RAG 对比**：微调适合知识内化，RAG 适合外部知识调用
- **场景选择**：专业领域（如医疗）、特定风格（如法律文书）
- **数据采集**：爬虫、公开数据集、企业内部数据

#### 2. 数据工程

- **去重**：去除重复样本
- **清洗处理**
  - 过滤：删除低质量内容（如乱码、不相关）
- **标注增强**
  - 人工标注：关键任务（如医疗问答）
  - 自动增强：同义词替换、句式转换

#### 3. 主流微调方法

##### LoRA（Low-Rank Adaptation）

- **原理**：冻结预训练模型，仅训练低秩矩阵
- **优势**：显著降低 FLOPs（7B 模型可在单卡训练）

##### QLoRA

- **优化**：量化预训练模型（4bit/8bit），进一步降低显存
- **适用场景**：消费级 GPU 训练（如 RTX 3090）

### （二）主流技术框架

- **HuggingFace Transformers**
  - 模型加载：`AutoModelForCausalLM.from_pretrained`
  - 训练器：`TrainingArguments` 配置超参数
- **Unsloth**
  - 特点：优化的 LoRA 实现，训练速度提升 2-5 倍
- **LLaMA-Factory**
  - 优势：支持多模型（LLaMA/Qwen 等）、多任务（SFT / Reward Model）

### （三）实战案例：医疗问诊助手

- **加载基础模型**：Qwen2-7B
- **配置 LoRA**：`r=8, lora_alpha=32`
- **训练参数**：`batch_size=4, epochs=3, learning_rate=2e-4`
- **合并权重**：LoRA 权重与基础模型合并
- **评估指标**：回答准确率（医疗专家评分）、相关性



##  七、大模型实战工具

### （一）本地化部署工具

#### Ollama/llama.cpp/vLLM
这三款工具正好对应了大模型部署的三个不同层次：**Ollama 追求“极简”，llama.cpp 死磕“极致”，而 vLLM 专注“极速”**。

为了方便你直观对比，我把它们的核心差异整理成了下面的表格：

| 对比维度 | ⚡ llama.cpp | 🐳 Ollama | 🚀 vLLM |
| :--- | :--- | :--- | :--- |
| **一句话定位** | 追求轻量与兼容的 **C++ 推理引擎** | 追求开箱即用的 **模型管理工具** | 追求极致吞吐的 **高性能服务框架** |
| **核心目标** | 让大模型在任何设备上都能运行 | 像用 Docker 一样简单地用大模型 | 在高并发下榨干 GPU 的每一份算力 |
| **典型用户** | 嵌入式/边缘计算工程师、极致性能调优者 | 开发者、研究者、想快速验证想法的人 | 运维工程师（SRE）、需要上线的 AI 应用团队 |
| **上手难度** | ⭐⭐⭐ 中等（需懂命令行） | ⭐ 极低（一条命令搞定） | ⭐⭐⭐ 中等（Python 环境，配置稍多） |
| **系统支持** | **全平台**（Win/Linux/macOS/手机/树莓派） | **全平台**（Win/Linux/macOS） | **仅 Linux**（对 Mac 和 Windows 不友好） |
| **硬件环境** | **CPU 优先**，也可用 GPU 加速 | CPU/GPU 均可（主要封装 llama.cpp） | **高性能 NVIDIA GPU** 是标配 |
| **核心技术** | GGUF 量化、ARM/AVX2 指令集优化 | 封装 llama.cpp，提供 HTTP API 和管理能力 | **PagedAttention**（类似虚拟内存的分页技术）、Continuous Batching（连续批处理） |
| **模型格式** | **GGUF**（专用格式，加载快） | GGUF（因为底层就是 llama.cpp） | Hugging Face 格式（如 FP16, BF16, GPTQ, AWQ） |
| **并发性能** | 一般（请求排队处理，适合单任务） | 较弱（封装层在高并发下易成瓶颈） | **极强**（吞吐量是 Ollama 的数倍甚至数十倍） |
| **最佳场景** | 老旧设备、树莓派、无独显环境、CPU 推理 | 个人开发者尝鲜、本地快速原型开发、集成到现有应用 | 企业级高并发 API 服务、大规模生产环境 |

---

#### 🎯 如何选型三步走

##### 1. 👨‍💻 个人开发者或想快速尝鲜 -> 首选 **Ollama**

如果你只是想在自己的 MacBook 或 Windows 电脑上本地跑个 Llama 3 或 Qwen 2.5 玩玩，或者快速验证一个想法，**Ollama 是绝对的首选**。它把下载、安装、运行都整合成了一条命令，还提供了和 OpenAI 几乎一样的 API 接口，你甚至可以把 `localhost:11434` 直接填进 Chatbox 或 NextChat 等开源的前端软件里，像使用 ChatGPT 一样使用本地模型，体验非常好。

##### 2. 💻 硬件配置有限，没有强力独显，甚至想部署在树莓派上 -> 首选 **llama.cpp**

如果你的环境是老旧的办公电脑、英特尔核显笔记本，甚至是想在工控机或树莓派上跑大模型，那 **llama.cpp 几乎是唯一的选择**。它能通过 GGUF 量化技术，把需要几十 G 显存的大模型压缩到只需要几个 G 的内存就能运行。虽然生成速度不如顶级显卡快，但能让大模型在更多设备上"活"起来。

##### 3. 🏢 企业级应用，需要服务成百上千用户，追求最大吞吐量 -> 首选 **vLLM**

如果你的场景是服务上线、需要支撑高并发，**vLLM 是当前工业界的事实标准**。凭借 PagedAttention 这项关键技术，vLLM 在高并发场景下的吞吐量远超前面两者，能最大化节省你的 GPU 算力成本。它被很多大厂用在生产环境中，稳定性经过了充分验证。

> 可能很多人不知道的是，其实 **Ollama 底层调用的正是 llama.cpp**。你也可以这么理解：Ollama ≈ 一个用 Go 写的、带 API 的、易用的"外壳" + 内部的 llama.cpp "核心"。因此，如果你在用 Ollama 时感觉参数不够灵活，本质上是因为 Ollama 没有把底层 llama.cpp 的全部能力暴露给你。

## 九、进阶：AI算法基础

### （一）机器学习核心

#### 1. 监督学习

- **回归算法**
  - 线性回归：房价预测案例
  - 逻辑回归：二分类（垃圾邮件识别）
- **KNN**：基于距离的分类
- **决策树**：可解释性强的分类模型
- **集成学习**：随机森林、XGBoost（提升准确率）

#### 2. 无监督学习

- **聚类算法**
  - K-means：用户分群案例
  - 密度聚类：异常检测（欺诈交易识别）
- **降维技术**
  - PCA：数据可视化（高维数据降为 2D/3D）

#### 3. 神经网络

- **基础结构**：输入层 → 隐藏层 → 输出层
- **激活函数**：
  - ReLU：解决梯度消失
  - Sigmoid：二分类输出
- **核心概念**：
  - 损失函数：MSE（回归）、交叉熵（分类）
  - 优化器：SGD、Adam（加速收敛）

### （二）深度学习基础

##### 1. CNN（卷积神经网络）

- **核心层**：
  - 卷积层：特征提取
  - 池化层：降维
- **应用**：图像分类（MNIST 手写数字识别）

#### 2. RNN（循环神经网络）

- **特点**：处理序列数据（文本/时间序列）
- **变体**：
  - LSTM：解决长依赖问题
  - GRU：简化版 LSTM

### （三）大模型训练技术

#### 1. 预训练流程

- **数据准备**：大规模文本 corpus（清洗、去重）
- **训练目标**：
  - 因果语言模型（CLM）
  - 编码语言模型（MLM）
- **分布式训练**：
  - 数据并行：多卡拆分数据
  - 模型并行：多卡拆分模型

#### 2. 微调与对齐

- **SFT（有监督微调）**：用高质量数据微调
- **RLHF（人类反馈强化学习）**：
  - **步骤 1**：奖励模型（RM）训练，标注回答质量作为奖励
  - **步骤 2**：RL 微调，用 PPO 算法最大化奖励
- **作用**：提升模型对任务的符合人类偏好
